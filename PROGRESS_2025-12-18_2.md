### 1. パスワード変更モーダルの修正
**コミット**: `e92b214`

#### 問題
管理画面の「パスワード変更」ボタンをクリックしても、モーダルが表示されない

#### 原因
`PasswordChangeModal` コンポーネントは `isOpen` プロップを必須としているが、`AdminDashboard.tsx` から渡されていなかった。

```typescript

```

そのため、`isOpen` が undefined となり、モーダルが何も表示しなかった。

#### 解決策
`AdminDashboard.tsx:372` に `isOpen={showPasswordChangeModal}` プロップを追加：

```typescript
<PasswordChangeModal
  isOpen={showPasswordChangeModal}
  user={user}
  onClose={() => setShowPasswordChangeModal(false)}
/>
```

### 4. loadAssignedShifts の外部キー参照エラーの修正
**コミット**: `c81518d`

#### 問題
AIマッチング結果が画面に表示されなくなった。コンソールに以下のエラーが表示される：

```
Error loading assigned shifts: Could not find a relationship between 'assigned_shifts' and 'pharmacist_id' in the schema cache
```

#### 原因
`useAdminData.ts` の `loadAssignedShifts` 関数（line 105-112）で、Supabaseの外部キー参照を使っていた：

```typescript
const { data: assignedData, error: assignedError } = await supabase
  .from('assigned_shifts')
  .select(`
    *,
    pharmacist:pharmacist_id(name),
    pharmacy:pharmacy_id(name)
  `)
  .order('created_at', { ascending: false });
```

しかし、Supabaseのスキーマキャッシュにこれらの関係が見つからなかった。

#### 解決策
外部キー参照を削除し、シンプルに `.select('*')` のみを使用：

```typescript
const { data: assignedData, error: assignedError } = await supabase
  .from('assigned_shifts')
  .select('*')
  .order('created_at', { ascending: false });
```

**備考**: 同じファイルの `reload` 関数（line 138-140）では既に `.select('*')` を使っていたため、こちらは正常に動作していた。
