### 4. シフト重複チェックの修正（1d16059, 2a87193, d5f069e）

#### Phase 1: 型定義の修正（1d16059）

**問題**: `creates`配列の型が`string[]`だが、実際には`{name: string, date: string}`オブジェクトを格納

**修正**:
```typescript
// 修正前
const creates: string[] = [];

// 修正後
const creates: { name: string, date: string }[] = [];
```

#### Phase 2: デバッグログの追加（2a87193）

**目的**: 同じ日付・店舗で異なる時間帯が「上書き」と判定される問題を調査

**追加したログ**:
- 既存シフトの詳細（date, store_name, time_slot, start_time, end_time）
- 新規登録の詳細（date, name, customTimeMode, startTime, endTime）
- 各比較ステップの結果（日付、薬局ID、時間帯、店舗名）
- 一致/不一致の判定理由

#### Phase 3: カスタム時間モードの設定（d5f069e）

**問題**: 定型時間テンプレートをクリックしても`customTimeMode`が`false`のまま

**原因**:
```
既存シフト: time_slot: "morning", start_time: "09:00", end_time: "13:00"
新規登録: time_slot: "morning", customTimeMode: false, startTime: "08:00", endTime: "20:00"
```
- テンプレートクリック時に`startTime`と`endTime`は変更されるが、`customTimeMode`は`false`のまま
- 重複チェックで`time_slot`（"morning"）のみを比較
- 異なる時間帯なのに「一致」と判定

**修正**:
```typescript
const applyTimeTemplate = (template: {name: string, start: string, end: string}) => {
  setStartTime(template.start);
  setEndTime(template.end);
  setCustomTimeMode(true); // ← 追加
};
```

**適用箇所**:
- `src/components/PharmacyDashboard.tsx`
- `src/components/PharmacistDashboard.tsx`

**効果**:
- 定型時間テンプレート使用時もカスタム時間として扱われる
- `start_time`と`end_time`を正しく比較
- 異なる時間帯は「新規」として作成される
