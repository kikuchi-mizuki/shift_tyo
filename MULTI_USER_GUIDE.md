# マルチユーザー同時利用ガイド

## 概要

シフト調整システムは、薬剤師、薬局、管理者が同時にシステムを利用できるマルチユーザー対応システムです。

## 同時利用可能なユーザータイプ

### 1. 薬剤師 (Pharmacist)
- **機能**: シフト希望の登録・更新・削除、確定シフトの確認、NG薬局設定
- **同時利用**: 複数の薬剤師が同時にログインしてシフト希望を登録可能
- **データ分離**: 各薬剤師のデータは独立して管理

### 2. 薬局 (Pharmacy)
- **機能**: 薬剤師募集の登録・管理、確定シフトの確認、NG薬剤師設定
- **同時利用**: 複数の薬局が同時にログインして募集を登録可能
- **データ分離**: 各薬局のデータは独立して管理

### 3. 管理者 (Admin)
- **機能**: シフトの一括確定・取り消し、ユーザー管理、マッチング管理
- **同時利用**: 管理者が他のユーザーと同時にログインして管理作業を実行可能
- **権限**: 全ユーザーのデータにアクセス可能

## 同時利用の特徴

### リアルタイム更新
- 各ユーザーの操作は即座にデータベースに反映
- 他のユーザーがページを更新すると最新の状態を確認可能
- オンラインユーザー表示で現在アクティブなユーザーを確認

### データ整合性
- Row Level Security (RLS) により適切な権限管理
- 各ユーザーは自分のデータのみアクセス可能
- 管理者のみ全データにアクセス可能

### セッション管理
- 各ユーザーが独立したセッションで動作
- ログアウトしても他のユーザーのセッションに影響なし
- 自動セッション更新で長時間の利用も可能

## 利用シナリオ

### シナリオ1: 薬剤師と薬局の同時利用
1. 薬剤師Aがシフト希望を登録
2. 薬局Bが同時に薬剤師募集を登録
3. 管理者Cがリアルタイムでマッチング状況を確認
4. 全員が同時にシステムを利用して効率的にシフト調整

### シナリオ2: 複数薬剤師の同時登録
1. 薬剤師A、B、Cが同時にログイン
2. 各薬剤師が独立してシフト希望を登録
3. 互いのデータに影響することなく操作可能
4. 管理者が一括でマッチング処理を実行

### シナリオ3: 薬局と管理者の連携
1. 薬局が薬剤師募集を登録
2. 管理者が同時にシステム状態を監視
3. 薬局の募集状況をリアルタイムで確認
4. 必要に応じて管理者がサポート

## 技術仕様

### 認証システム
- Supabase Auth による安全な認証
- JWT トークンベースのセッション管理
- 自動トークン更新

### データベース
- PostgreSQL with Row Level Security
- リアルタイム更新対応
- 適切なインデックス設定

### フロントエンド
- React with TypeScript
- 遅延読み込みによる最適化
- エラーハンドリングとリトライ機能

## 注意事項

### 同時編集の制限
- 同じデータを複数ユーザーが同時編集する場合は、最後の更新が優先
- 重要な操作（シフト確定など）は管理者のみ実行可能

### パフォーマンス
- 同時接続数に制限はありませんが、大量の同時操作時はレスポンス時間が延長する可能性
- 定期的なページ更新で最新状態を確認

### セキュリティ
- 各ユーザーは自分のデータのみアクセス可能
- 管理者権限は適切に管理
- すべての操作はログに記録

## トラブルシューティング

### よくある問題

**Q: 他のユーザーの操作が反映されない**
A: ページを更新してください。リアルタイム更新は部分的に実装されています。

**Q: ログインできない**
A: 他のユーザーが同じアカウントでログインしていないか確認してください。

**Q: データが消えた**
A: 管理者に確認してください。RLSにより適切にデータが保護されています。
