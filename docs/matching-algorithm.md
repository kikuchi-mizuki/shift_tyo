## マッチングとは？

**マッチング**とは、「薬剤師の希望」と「薬局の募集」を最適に組み合わせることです。

### 例えば...

```
薬剤師A: 3月10日、9時〜18時で働きたい
薬剤師B: 3月10日、13時〜19時で働きたい
薬剤師C: 3月10日、9時〜15時で働きたい

薬局X: 3月10日、9時〜18時で1名募集
薬局Y: 3月10日、14時〜19時で1名募集
薬局Z: 3月10日、9時〜13時で1名募集
```

この場合、どの薬剤師をどの薬局に割り当てるのが最適でしょうか？

AIマッチングアルゴリズムが、**自動的に最適な組み合わせ**を見つけます。

---

## アルゴリズムの目的

AIマッチングアルゴリズムは、以下の3つの目的を達成します：

### 🎯 1. 不足薬局をゼロに近づける

できるだけ多くの薬局に薬剤師を割り当てます。

**悪い例:**
```
薬局A: 募集1名 → 割り当て1名 ✅
薬局B: 募集1名 → 割り当て0名 ❌（不足！）
薬局C: 募集1名 → 割り当て0名 ❌（不足！）
```

**良い例:**
```
薬局A: 募集1名 → 割り当て1名 ✅
薬局B: 募集1名 → 割り当て1名 ✅
薬局C: 募集1名 → 割り当て1名 ✅
```

### 📍 2. 距離を考慮する

薬剤師が通いやすい薬局を優先します。

```
薬剤師A（渋谷在住）
  → 薬局X（渋谷駅近く）: 距離スコア 90点 ⭐⭐⭐
  → 薬局Y（横浜駅近く）: 距離スコア 40点 ⭐
```

### ⚖️ 3. 公平性を保つ

特定の薬剤師ばかりが選ばれないよう、応募回数も考慮します。

```
薬剤師A: 今月0回応募 → 優先度 高 ⬆️
薬剤師B: 今月5回応募 → 優先度 低 ⬇️
```

## マッチングの流れ

マッチングは**5つのステップ**で行われます：

```
┌─────────────────────────────────────┐
│ ステップ1: データの準備              │
│  - 薬剤師の希望リスト                │
│  - 薬局の募集リスト                  │
│  - 確定済みシフトを除外              │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ ステップ2: 互換性チェック            │
│  - 時間が合うか？                    │
│  - NGリストに入っていないか？        │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ ステップ3: スコア計算                │
│  - 距離スコア（40%）                 │
│  - 応募回数スコア（30%）             │
│  - 評価スコア（30%）                 │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ ステップ4: 最適化マッチング          │
│  - 候補が少ない薬局から優先処理      │
│  - スコアが高い薬剤師を割り当て      │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ ステップ5: 再配置による改善          │
│  - 不足がある薬局に薬剤師を再配置    │
│  - 全体の不足を最小化                │
└─────────────────────────────────────┘
                 ↓
          【マッチング完了】
```

## ステップ1: データの準備

### やること

その日の「薬剤師の希望」と「薬局の募集」を集めます。

### 除外されるデータ

✅ **含まれる:**
- ステータスが「pending（未確定）」の希望・募集
- 確定していない薬剤師・薬局

❌ **除外される:**
- すでに確定したシフト
- その日に別の薬局で確定している薬剤師
- 必要人数が満たされている薬局

### 例

**3月10日のデータ:**

| 薬剤師 | 希望時間 | ステータス | マッチング対象？ |
|--------|----------|------------|------------------|
| 薬剤師A | 09:00-18:00 | pending | ✅ Yes |
| 薬剤師B | 09:00-18:00 | confirmed | ❌ No（すでに確定） |
| 薬剤師C | 13:00-19:00 | pending | ✅ Yes |

| 薬局 | 募集時間 | 必要人数 | 確定人数 | マッチング対象？ |
|------|----------|----------|----------|------------------|
| 薬局X | 09:00-18:00 | 1名 | 0名 | ✅ Yes |
| 薬局Y | 14:00-19:00 | 2名 | 2名 | ❌ No（満員） |
| 薬局Z | 09:00-13:00 | 1名 | 0名 | ✅ Yes |

## ステップ2: 互換性チェック

### やること

「薬剤師の希望」と「薬局の募集」が**合うかどうか**をチェックします。

### チェック項目

#### 1️⃣ 時間が100%カバーしているか？

薬剤師の希望時間が、薬局の募集時間を**完全に包含**している必要があります。

```
✅ OK: 薬剤師の希望が薬局の時間を完全にカバー
薬剤師: ━━━━━━━━━━━━━━━  (09:00-18:00)
薬局:     ━━━━━━━━━━      (10:00-17:00)

❌ NG: 終了時間が足りない
薬剤師: ━━━━━━━━━━━      (09:00-16:00)
薬局:     ━━━━━━━━━━━━  (10:00-18:00)

❌ NG: 開始時間が遅い
薬剤師:       ━━━━━━━━━  (11:00-18:00)
薬局:     ━━━━━━━━━━━━  (09:00-17:00)
```

#### 2️⃣ NGリストに登録されていないか？

- **薬剤師側のNGリスト**: この薬局では働きたくない
- **薬局側のNGリスト**: この薬剤師には来てほしくない

```
❌ NG: 薬剤師Aが薬局Xを拒否している
❌ NG: 薬局Yが薬剤師Bを拒否している
```

### 互換性の判定

```typescript
互換性あり = 時間が100%カバー AND NGリストに入っていない
```

## ステップ3: スコア計算

互換性がある組み合わせに対して、**どれくらい良い組み合わせか**をスコアで評価します。

### スコアの内訳

| 項目 | 重み | 説明 |
|------|------|------|
| **距離スコア** | 40% | 薬剤師の最寄駅と薬局の距離 |
| **応募回数スコア** | 30% | 今月の応募回数（少ない方が優先） |
| **評価スコア** | 30% | 薬剤師の評価（5段階） |

### 計算式

```
総合スコア = (距離スコア × 0.4) + (応募回数スコア × 0.3) + (評価スコア × 0.3)
```

### 具体例

**薬剤師A → 薬局X**

```
距離スコア: 0.9（同じ駅で非常に近い）
応募回数スコア: 0.8（今月1回応募）
評価スコア: 1.0（5.0 / 5.0 = 満点）

総合スコア = (0.9 × 0.4) + (0.8 × 0.3) + (1.0 × 0.3)
           = 0.36 + 0.24 + 0.30
           = 0.90（90点）⭐⭐⭐
```

**薬剤師B → 薬局X**

```
距離スコア: 0.4（遠い）
応募回数スコア: 0.3（今月5回応募）
評価スコア: 0.6（3.0 / 5.0）

総合スコア = (0.4 × 0.4) + (0.3 × 0.3) + (0.6 × 0.3)
           = 0.16 + 0.09 + 0.18
           = 0.43（43点）⭐
```

**結果**: 薬剤師Aの方がスコアが高いので、薬局Xには薬剤師Aが優先的に割り当てられます。

## ステップ4: 最適化マッチング

### 基本戦略: 難しい薬局から先に処理

**「候補が少ない薬局」から順番に処理**します。

#### なぜ？

候補が多い薬局は後回しにしても大丈夫ですが、候補が少ない薬局を後回しにすると、誰もいなくなってしまう可能性があるからです。

### 処理の流れ

```
1. 各薬局の候補者数をカウント
   薬局A: 候補3人
   薬局B: 候補1人（少ない！）
   薬局C: 候補5人

2. 候補が少ない順に並べ替え
   薬局B（1人）→ 薬局A（3人）→ 薬局C（5人）

3. 順番に最適な薬剤師を割り当て
   薬局B: スコア1位の薬剤師Xを割り当て ✅
   薬局A: スコア1位の薬剤師Yを割り当て ✅
   薬局C: スコア1位の薬剤師Zを割り当て ✅
```

### 具体例

**3月10日のマッチング:**

```
薬局リスト（候補者数でソート）:
1. 薬局Y（候補1人）← 最優先で処理
2. 薬局X（候補3人）
3. 薬局Z（候補5人）← 最後に処理

処理開始:
┌─────────────────────────────────┐
│ 薬局Y（候補1人、必要1名）       │
│ 候補: 薬剤師C（スコア80点）     │
│ → 薬剤師Cを割り当て ✅          │
└─────────────────────────────────┘
   ↓ 薬剤師Cは使用済み
┌─────────────────────────────────┐
│ 薬局X（候補3人、必要1名）       │
│ 候補: 薬剤師A（90点）           │
│       薬剤師B（43点）           │
│       薬剤師C（80点）← 使用済み │
│ → 薬剤師Aを割り当て ✅          │
└─────────────────────────────────┘
   ↓ 薬剤師A, Cは使用済み
┌─────────────────────────────────┐
│ 薬局Z（候補5人、必要1名）       │
│ 候補: 薬剤師D（85点）← 最高    │
│       薬剤師E（70点）           │
│       その他...                 │
│ → 薬剤師Dを割り当て ✅          │
└─────────────────────────────────┘

結果: 全3店舗に割り当て成功！不足ゼロ 🎉
```

## ステップ5: 再配置による改善

### やること

ステップ4で不足が残った場合、**薬剤師を再配置**して不足を減らします。

### どういう時に再配置する？

```
状況: 薬局Aに薬剤師が余っている、薬局Bが不足している

薬局A（必要1名）: 薬剤師X, 薬剤師Y が互換性あり
  → 現在: 薬剤師Xが割り当て済み ✅
  → 余裕: 薬剤師Yも使える

薬局B（必要1名）: 薬剤師Xのみ互換性あり
  → 現在: 誰も割り当てられていない ❌

解決策: 薬剤師Xを薬局Bに移動
  薬局A: 薬剤師X → 薬剤師Y に変更
  薬局B: なし → 薬剤師X に変更
  結果: 不足が1つ減る！
```

### 再配置の条件

再配置は以下の条件を満たす場合のみ実行されます：

✅ **再配置OK:**
- 元の薬局に他の候補者がいる
- 移動先の薬局にその薬剤師が互換性がある

❌ **再配置NG:**
- 元の薬局の候補者が1人だけ（移動すると元が不足になる）
- 移動先の薬局と時間が合わない

### 処理フロー

```
最大10回繰り返し:
  1. 不足がある薬局をリストアップ
  2. その薬局に割り当て可能な薬剤師を探す
  3. その薬剤師が現在別の薬局に割り当てられているか確認
  4. 元の薬局に余裕があれば、再配置を実行
  5. 不足が減ったら次の薬局へ
  6. すべての不足が解消されたら終了
```

## インタラクティブマッチング

### 概要

管理者が**手動で薬剤師を入れ替える**ことができる機能です。

```
AIの提案:
  A薬局 → 田中さん
  B薬局 → 佐藤さん
  C薬局 → 高橋さん

管理者の判断:
  「C薬局は高橋さんより佐藤さんの方がいい」
  → ドロップダウンで変更

結果（自動再最適化）:
  A薬局 → 田中さん（変更なし）
  B薬局 → 高橋さん（自動で変更）
  C薬局 → 佐藤さん（手動で変更）✅
```

### 特徴

#### 🔒 固定割り当て

手動で変更した薬局は**固定**され、再最適化時も変更されません。

```
固定: C薬局 → 佐藤さん（管理者が選択）
↓
他の薬局を自動再最適化
↓
C薬局の割り当ては維持される
```

#### 🔄 自動再最適化

手動で変更すると、他の薬局のマッチングが**自動的に再計算**されます。

```
手動変更前:
  A薬局 → 田中さん
  B薬局 → 佐藤さん
  C薬局 → 高橋さん

C薬局を佐藤さんに変更:
  C薬局 → 佐藤さん（固定）✅
  ↓
  自動再計算:
    A薬局 → 田中さん（変更なし）
    B薬局 → 高橋さん（自動変更）✅
```

#### 📋 候補者リスト

各薬局に割り当て可能な全薬剤師がドロップダウンで表示されます。

```
A薬局のドロップダウン:
  田中さん（91点）← 現在の割り当て
  ---
  （他に互換性のある薬剤師はいません）

B薬局のドロップダウン:
  佐藤さん（61点）← 現在の割り当て
  高橋さん（88点）(現在: C薬局)
  田中さん（77点）(現在: A薬局)
```

## まとめ

### アルゴリズムの強み

✅ **不足を最小化**: 候補が少ない薬局から優先処理
✅ **距離を考慮**: 薬剤師が通いやすい薬局を優先
✅ **公平性**: 応募回数が少ない薬剤師を優先
✅ **柔軟性**: 手動での微調整が可能
✅ **タイムゾーン対応**: 月末日も正しく処理

### 処理の流れ（再掲）

```
1. データ準備 → 2. 互換性チェック → 3. スコア計算
                      ↓
4. 最適化マッチング（候補が少ない順）
                      ↓
5. 再配置による改善（不足を減らす）
                      ↓
      【マッチング完了】
```
