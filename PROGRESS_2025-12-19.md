### 5. マッチング済み薬局が不足薬局に表示される問題の修正
**コミット**: `5e275ac`

#### 問題
AIマッチングが完了しているのに、同じ薬局が「不足薬局」セクションにも表示される

#### 原因
`analyzePharmacyShortage`関数が`aiMatchesByDate`のみを見ており、`assigned`テーブルのデータを見ていなかった：
- `aiMatchesByDate`はメモリ上にのみ存在
- ページリロード後は空になる
- 実際のマッチングデータは`assigned`テーブルに保存されている（status='pending'）
- 結果：`matched=0` → 不足薬局として表示

#### 解決策
**ファイル**: `src/services/admin/AnalysisService.ts:289-300, 350-368`

```typescript
// 🔧 FIX: aiMatchesByDateが空でも、assignedテーブルからpendingマッチングを取得
const pendingMatches = Array.isArray(assigned)
  ? assigned.filter((s: any) => s?.date === date && s?.status === 'pending')
  : [];

// aiMatchesByDateが空の場合は、assignedから取得したpendingマッチングを使用
const effectiveDayMatches = dayMatches.length > 0 ? dayMatches : pendingMatches;
```

**効果**:
- Before: `matched=0` (incorrect) → 不足薬局として表示 ❌
- After: `matched=1` (correct) → 不足薬局に表示されない ✅
