### 2. 薬局募集管理機能の改善
**コミット**: `ba84a43`

#### 問題
1. 同じ日付で異なる時間帯の募集ができない（午前と午後を別々に募集したい）
2. 個別の募集を編集・削除できない（一括でしか操作できない）

#### 解決策

##### 2.1 複数時間帯の募集対応
**ファイル**: `src/components/PharmacyDashboard.tsx` (1208-1248行目)

**変更内容**:
```typescript
// 修正前: 日付と店舗のみで重複判定
const match = myShifts.find((s: any) =>
  s.date === date && s.store_name === name
);

// 修正後: 日付 + 店舗 + 時間帯で重複判定
const match = myShifts.find((s: any) => {
  if (s.date !== date) return false;
  if (s.pharmacy_id !== user?.id) return false;

  // 時間帯も比較
  const currentTimeSlot = customTimeMode ? 'fullday' : timeSlot;
  if (customTimeMode && s.time_slot === 'fullday') {
    // カスタム時間の場合は start_time/end_time も比較
    const sStart = s.start_time ? String(s.start_time).substring(0, 5) : '';
    const sEnd = s.end_time ? String(s.end_time).substring(0, 5) : '';
    if (sStart !== startTime || sEnd !== endTime) return false;
  } else {
    if (s.time_slot !== currentTimeSlot) return false;
  }

  if (s.store_name !== name) return false;
  return true;
});
```

**効果**:
- ✅ 同じ日付で午前（09:00-13:00）と午後（13:00-18:00）を別々に募集可能
- ✅ カスタム時間帯も複数登録可能
- ✅ 時間帯が異なれば重複エラーにならない

## 技術的詳細

### パスワードリセットフロー
```
1. ユーザーが「パスワードを忘れた方」をクリック
   ↓
2. メールアドレスを入力
   ↓
3. Supabase Auth がリセットメールを送信
   ↓
4. メールのリンクをクリック
   ↓
5. /reset-password にリダイレクト（トークン付き）
   ↓
6. 新しいパスワードを入力・確認
   ↓
7. supabase.auth.updateUser() でパスワード更新
   ↓
8. 自動的にログイン画面へリダイレクト（3秒後）
```

### 募集重複チェックロジック
```typescript
重複判定の条件:
- 日付が一致 AND
- 店舗が一致 AND
- 時間帯が一致 AND
- カスタム時間の場合は start_time/end_time も一致

→ すべて一致する場合のみ「重複」として扱う
→ 時間帯が異なれば別の募集として登録可能
```

### 定型時間テンプレートのデータ構造
```typescript
// localStorage に保存される形式

  value: [

    { name: "午後", start: "13:00", end: "18:00" },
    { name: "夜勤", start: "18:00", end: "22:00" }
  ]
}
```

## 主要な設計判断

### 1. パスワードリセット
**判断**: メール経由のリセットフローを採用

**理由**:
- セキュリティ: Supabase Authの標準機能を使用
- UX: 一般的なWebサービスと同じフロー
- 実装コスト: 既存の認証システムと統合しやすい

### 2. 募集の重複判定
**判断**: 日付 + 店舗 + 時間帯の3要素で判定

**理由**:
- 柔軟性: 同じ日付でも時間帯が違えば別の募集
- 明確性: 完全一致の場合のみ重複とする
- 拡張性: カスタム時間にも対応

### 3. 定型時間の保存先
**判断**: localStorageを使用（データベースではなく）

**理由**:
- 実装速度: すぐに使える
- パフォーマンス: サーバー通信不要
- プライバシー: ローカルのみに保存
- 十分性: テンプレートは個人設定であり共有不要
