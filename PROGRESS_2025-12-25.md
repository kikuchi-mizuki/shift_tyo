### 1. 薬局ダッシュボードの募集バッジ表示問題（続き）
**コミット**: `884fc9a`, `6d12b7b`

#### 問題
前回のセッション（12月24日）で管理者ダッシュボードの募集バッジ表示を修正したが、薬局ダッシュボードでも同じ問題が発生していた。2人募集で1人確定した後、募集バッジが消えてしまう。

#### 原因
**PharmacyDashboard.tsx**で以下の2つの問題が発生:
1. **データフィルタリング**: 4箇所で `status !== 'confirmed'` によるフィルタリングが実行され、部分確定後の募集が非表示になっていた
2. **カレンダー表示ロジック**: 確定シフトが1件でもある日付は、募集バッジを非表示にしていた

#### 解決策

##### 1.1 募集フィルタリングの削除 (884fc9a)
**ファイル**: `src/components/PharmacyDashboard.tsx`

**変更箇所**:
- loadData関数 (298-308行目)
- handlePost関数 (1307-1312行目)
- handleUpdateExisting関数 (1409-1413行目)
- handleDeleteExisting関数 (1446-1455行目)

```typescript
// 修正前
const filteredPostings = (myShiftsData || []).filter((posting: any) => {
  return posting.status !== 'confirmed';
});
setMyShifts(filteredPostings);

// 修正後
// 募集は常に表示する（required_staffが2以上の場合、部分確定でも募集継続のため）
setMyShifts(myShiftsData || []);
```

##### 1.2 カレンダーバッジロジックの修正 (6d12b7b)
**ファイル**: `src/components/PharmacyDashboard.tsx` (1598-1626行目)

```typescript
// 修正前
const hasConfirmed = confirmedShifts.some((s: any) => s.date === dateStr);
if (hasConfirmed) return null; // ❌ 確定があれば募集バッジを非表示

// 修正後
// 募集ごとに必要人数と確定人数を比較
let shortageCount = 0;
postingsForDate.forEach((posting: any) => {
  const requiredStaff = posting.required_staff || posting.required_people || 1;
  const confirmedCount = confirmedForDate.filter((confirmed: any) =>
    confirmed.time_slot === posting.time_slot &&
    confirmed.pharmacy_id === posting.pharmacy_id
  ).length;

  if (requiredStaff > confirmedCount) {
    shortageCount++; // ✅ 不足がある場合のみカウント
  }
});
```

**効果**:
- ✅ 2人募集で1人確定 → 「募集 1件」を表示
- ✅ 2人募集で2人確定 → バッジ非表示
- ✅ 既存データでも正しく動作

## 技術的詳細

### 主要な設計判断

#### 1. 募集ステータス管理の方針
```
【従来の問題のある設計】
shift_postings.status = 'confirmed' に更新 → フィルタで除外 → 非表示

【改善後の設計】
shift_postings.status は更新しない → すべて表示 → required_staff vs 確定数で判定
```

**利点**:
- 部分確定時の募集継続を正確に表現
- データベース更新が不要（パフォーマンス向上）
- ロジックがシンプルで保守しやすい

#### 2. UIレベルでのデータ保護
```typescript
// バックエンドとフロントエンド両方でチェック

// フロントエンド（UI）
if (hasConfirmedShifts) {
  return <ConfirmedMessage />; // ボタンを非表示
}

// バックエンド（バリデーション）
if (confirmedDates.length > 0) {
  alert('確定済みのため編集できません');
  return;
}
```

**利点**:
- ユーザーに即座にフィードバック（UX向上）
- 多重防御（セキュリティ向上）
- 誤操作を防止
