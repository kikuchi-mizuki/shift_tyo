# 店舗名表示問題の修正手順

## 問題の概要

管理画面の薬局一覧で店舗名が表示されない問題が発生していました。

### 原因

データベースの`user_profiles`テーブルの`store_names`カラム（配列型）が空配列になっており、店舗名データが登録されていませんでした。

### 調査結果

```
📊 全ユーザープロファイル取得: 43
薬局データ: 14件

ほとんどの薬局で store_names: Array(0) （空配列）
例外: テストアカウントのみ store_names: Array(1)
```

## マイグレーションの詳細

### 抽出ロジック

1. **括弧内の名前を優先**
   - `有限会社フラワーズ(フラワー)` → `フラワー`
   - `株式会社Re-pharame(つばさ薬局)` → `つばさ`

2. **会社形態のプレフィックスを削除**
   - `株式会社グラム` → `グラム`
   - `有限会社メイプル薬局` → `メイプル`

3. **「薬局」サフィックスを削除**
   - `株式会社ノムラ薬局` → `ノムラ`
   - `岡田薬局` → `岡田`

4. **そのまま使用**
   - `神明堂` → `神明堂`

### 対象データ

- `user_type` が 'pharmacy' または 'store'
- `store_names` が NULL、空配列、または未設定
- `name` が NULL でない

### 安全性

- 既存の`store_names`データは上書きされません
- `name`フィールドは変更されません
- ロールバック可能（必要に応じて手動で`store_names`をNULLに設定）

## トラブルシューティング

### 問題: マイグレーション実行後も店舗名が表示されない

**確認事項:**
1. ブラウザのハードリロード（Ctrl+Shift+R / Cmd+Shift+R）
2. ブラウザコンソールで`store_names`を確認
3. Supabaseで直接データを確認:
   ```sql
   SELECT id, name, store_names
   FROM user_profiles
   WHERE user_type IN ('pharmacy', 'store');
   ```

### 問題: 店舗名の抽出結果が期待と異なる

**対処法:**
1. 特定の薬局の`store_names`を手動で修正:
   ```sql
   UPDATE user_profiles
   SET store_names = ARRAY['正しい店舗名']
   WHERE id = '薬局のID';
   ```

2. または、管理画面から編集機能を使用

## 今後の運用

### 新規薬局登録時

新しい薬局を登録する際は、`store_names`配列に店舗名を設定してください。

### 店舗名の更新

管理画面の薬局一覧から「編集」ボタンで店舗名を追加・変更できます。
カンマ区切りで複数の店舗名を入力可能:
```
渋谷,新宿,池袋
```
