### 3. AIマッチングが特定日付のみ動作する問題の調査と修正

#### 問題の発見:
11月29日のみマッチングされ、他の日付でマッチングが作成されない

#### 根本原因:
**MatchingService.ts line 428**: `time_slot`のみのシフト希望/募集が除外されていた

```typescript
// 問題のコード
.filter((r: any) => r.start_time && r.end_time)
```

薬剤師・薬局が登録するデータは通常`time_slot`（morning/afternoon/fullday）のみで、
`start_time`/`end_time`が未設定のため、全てフィルターで除外されていた。

#### 修正内容:
**コミット**: `0a5a2a1`, `1b0f687`

**ファイル**: `src/services/admin/MatchingService.ts`

1. `normalizeTime`関数を追加:
```typescript
const normalizeTime = (request: any) => {
  if (request.start_time && request.end_time) {
    return request;
  }

  const timeSlotMap = {
    'morning': { start: '09:00:00', end: '13:00:00' },
    'afternoon': { start: '13:00:00', end: '18:00:00' },
    'fullday': { start: '09:00:00', end: '18:00:00' },
    'negotiable': { start: '09:00:00', end: '18:00:00' }
  };

  const timeSlot = request.time_slot || 'fullday';
  const times = timeSlotMap[timeSlot] || timeSlotMap['fullday'];

  return {
    ...request,
    start_time: times.start,
    end_time: times.end
  };
};
```

2. シフト希望と募集の両方に適用:
```typescript
// 希望側
const sortedRequests = filteredRequests
  .map(normalizeTime)
  .filter((r: any) => r.start_time && r.end_time)
  .sort(...);

// 募集側
const pharmacyNeeds = filteredPostings.map((p: any) => {
  const normalized = normalizeTime(p);
  return {
    ...normalized,
    remaining: Number(p.required_staff) || 0
  };
});
```
